一、从HSTU看生成式推荐：范式改变

在阅读生成式推荐（如 HSTU）的相关工作后，传统推荐系统更多依赖于特征工程与点式建模（point-wise），其本质是对“用户-物品匹配关系”的静态建模，这种方式难以刻画用户行为的动态演化过程。

HSTU 的关键突破在于，将推荐问题重新表述为一个序列生成问题，即通过自回归方式建模用户行为序列。这一点与大语言模型的建模方式高度一致，本质上是将“用户历史”视为上下文，将“未来行为”视为预测目标。在这一框架下，推荐系统不再是一个静态打分器，而是一个具备时间因果结构的生成模型。

这种转变的核心意义在于引入了正确的归纳偏置（inductive bias）。自回归结构通过 causal mask 强制模型遵循“过去影响未来”的因果关系，从而避免模型学习到无意义的统计相关性（例如未来行为影响过去行为）。这一点在推荐场景中尤为重要，因为数据本身高度稀疏且存在长尾分布，如果缺乏因果约束，模型很容易陷入过拟合和记忆化。但这种架构的本身也解放了特征工程，释放了系统资源，为处理更大规模候选集创造了可能。

实时特征的分析，和“历史行为的组合”成了新的问题导向。

重点思考：从 Action-Oriented Mask 到“结构化 Gating Attention”

在这一节中，我认为最有意思、也最值得深入思考的，并不是“序列长度减半”这一显式优化，而是其背后的 mask 设计思想。

它通过 mask 机制重新定义了信息在模型中的流动方式：每一个 candidate 都可以完整地访问用户历史信息，但不同 candidate 之间却被严格隔离、互不可见。这种设计乍看只是为了避免信息干扰，它就像是在 Transformer 的 attention 机制中显式地引入了一种带有 gating 性质的注意力机制。

传统的 self-attention 是一个完全对称且全连接的信息交互过程，所有 token 默认都可以互相通信，而这里通过 mask 的人为限制，将 attention 从“全连接图”变成了一个具有明确结构的有向图：历史 token 之间遵循因果结构，candidate token 可以读取历史，但 candidate 之间不存在边。

通过这种方式可以实现新的信息流通：
attention 不再只是一个用来做特征交互的计算工具，而是可以被视为一种“可编程的信息流控制机制”，通过设计不同的 mask 结构，我们实际上是在定义模型内部的信息传播路径，从而显式注入任务先验。例如在多模态 RAG 中，可以限制不同模态之间的信息交互方式以避免噪声传播；在多智能体系统中，可以让多个 agent 共享全局记忆但禁止直接相互干扰；在图结构任务中，甚至可以直接用图的邻接关系作为 attention mask 来替代传统的 message passing。这种想法十分有趣

6.2.3. 位置与时间：该学习什么、该编码什么

我们刚刚意识到GenRank是在序列建模中，去找究竟哪些信息必须通过参数学习获得，哪些信息可以直接编码进去；

HSTU 采用的是一种“全学习”的思路，通过相对注意力偏置（RAB）让模型同时学习位置差异、时间差异甚至行为类型之间的复杂关系，但这种方式带来了显著的计算和存储开销（O(N²)），尤其在长序列场景下会受到内存带宽的严重限制。

而 GenRank 的关键洞察在于，将时间与位置信息拆解为“绝对信息”和“相对信息”：前者（如序列位置、session划分、请求间隔）具有结构性和语义性，需要通过embedding学习；后者（如距离越远影响越小）则是一种普适的先验规律，可以用无参数的线性bias（类似ALiBi）直接编码。这种设计本质上是在用“规则+学习”的混合方式替代纯学习机制，从而在大幅降低计算复杂度的同时，反而提升了模型稳定性和泛化能力。但是GenRank 放弃了传统推荐系统中大量依赖交叉特征的建模方式，这在大规模数据场景下是合理的，因为模型可以自动学习这些关系，但在资源受限或特征工程高度成熟的场景中，可能会损失一部分性能。

但是，我们可以看到的是这种模式也在部分拆解原本的scaling Law；我们可以通过对问题本身进行了合理的结构分解，将“该学习的部分”和“无需学习的先验”清晰地区分开来。所以这种简化也给我们带来了新的思考：用户粒度的序列建模优势，是否必须依赖于完全的生成式范式？
