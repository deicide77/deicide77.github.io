5.4. 推荐中的Tokenizer技术

本节将探讨的是：如何将推荐系统中的物品转化为生成式模型能够理解和生成的Token序列？

5.4.1. Tokenizer范式演进

5.4.1.1. 三种范式的技术特征

稀疏ID范式（Sparse ID-Based）

这是传统推荐中的典型范式，将每个物品表示为一个唯一的离散ID
然后将每个ID映射到连续空间，最终模型学习用户embedding和item embedding的关系；
首先这种ID本质上不符合生成模型的建模假设（不存在语义），其次对于物品的embedding此处只能算是统计共现也就是传统的数据分析，不是语义理解；所以这种方法在判别式推荐中效果很好，但在生成式模型中面临词表爆炸、无法泛化和语义缺失等根本性问题。

三大根本困境

词表爆炸问题：首先稀疏ID本省每个都类似一个One-hot值，这就意味着如果将每个物品ID视为一个词，抖音的数十亿视频、淘宝的数亿商品，词表大小将达到十亿级别

存储与泛化的双重困境：十亿个ID存储是要TB级存储，但是原子ID之间完全正交，模型无法对新物品有泛化能力（冷启动问题）

协同信号的隐式依赖：强烈依靠大数据量分析，在数据稀疏场景下效果堪忧

优势

当然这种方法也有它的优势，由于基本范式还是传统推荐的范式，所以可以直接匹配当前的推荐系统，其次这种不包含语义的ID我们可以涉及多种特征组合网络；

解释：特征交互 = 不同特征之间“如何一起影响预测结果”
稀疏ID由于每个都直接embedding作为向量，所以可以进行拼接、点积、注意力，自由学习组合关系；以DIN举例：

  • DIN（Deep Interest Network）公式与示例
    给定： 用户历史行为序列：[A, B, C] 候选物品：D
    目标： 预测用户是否会点击 D
    过程：

    • 将所有ID映射为向量：
      e_A, e_B, e_C ∈ R^d （用户历史行为）
      e_D ∈ R^d （候选物品）

    • 对于每个历史行为 i（A, B, C），计算其对当前物品 D 的重要性：
      α_i = softmax( f(e_i, e_D) )
      其中：
      f(e_i, e_D) = MLP( [e_i, e_D, e_i - e_D, e_i ⊙ e_D] )

    • 用户兴趣表示
      user_interest = Σ (α_i * e_i)

    • 将用户兴趣和候选物品一起输入：
      y = σ( MLP( [user_interest, e_D] ) )
      其中：

      • σ 是 sigmoid 函数
      • 输出 y ∈ [0,1]，表示点击概率

DIN 通过 Attention 机制，对用户历史行为进行加权，使用户兴趣表示依赖于当前候选物品，从而实现了动态特征交互建模。这体现了稀疏ID范式中“特征交互自由”的核心优势。
