第五章 生成式推荐基础前言

生成式推荐研究的三种主要路径

在初读第五章开头时，我其实产生了一个比较明显的感受：作者试图先给读者建立一张“研究地图”。
也就是说，在真正进入具体算法之前，先说明当前生成式推荐研究大致沿着哪些方向发展。

然而，仅仅阅读教程中的这些名词时，我发现自己虽然具备一定的深度学习基础，但仍然很难清晰地解释这些概念之间的关系。因此，在查阅一些相关论文和资料之后，我对这一部分进行了简单的整理。

目前围绕 生成式推荐（Generative Recommendation） 的研究，大致可以分为三条主要路径：

  • 渐进式演进路径（Progressive Evolution）
  • 知识增强路径（Knowledge-enhanced Recommendation）
  • 完全生成式路径（Fully Generative Recommendation）

这三种路径并不是互相替代的关系，而更像是从不同角度探索生成式推荐的可能性。

渐进式演进路径

在教程中提到的第一条研究路线是 渐进式演进路径（Progressive Evolution）。
在初读这一部分时，我其实产生了一个疑问：这里提到的 Scaling 思想，是否只是简单地“把模型变大”？

最开始我的理解是：推荐系统借鉴大型语言模型的发展经验，通过扩大模型规模来提升推荐能力。例如：

  • 增大 embedding 维度
  • 增加网络深度
  • 提升模型参数规模

但在进一步查阅相关论文之后，我逐渐意识到，这里的“Scaling”并不仅仅指模型规模的扩大，还有 模型结构的升级。

在传统推荐系统中，排序模型通常比较简单，例如：

模型的核心任务仍然是计算一个匹配分数：
Score = f(User, Item, Context)

这种模型虽然在工程上非常高效，但它通常只能学习 静态的用户兴趣表示。

然而，如果观察真实用户行为，就会发现用户行为天然具有 序列结构。例如，一个用户的浏览行为可能是：

手机 → 手机壳 → 蓝牙耳机 → ?

这些行为之间往往具有明显的时间顺序和语义关联。用户的兴趣并不是静态的，而是随着行为不断演化的。

因此，一些研究开始尝试将 用户行为建模为序列数据，并使用更强的模型来学习用户兴趣的演化过程。例如：

  • 使用 RNN / GRU 建模行为序列
  • 使用 Transformer 或自注意力机制 捕捉长期依赖关系
  • 扩大模型规模，提高表示能力

典型研究包括：

  • SASRec（Self-Attentive Sequential Recommendation）
  • BERT4Rec
  • DIN / DIEN
  • 各类 Transformer-based 推荐模型

读到这里时，我其实产生了一个新的困惑：
如果用户表示已经从User ID 变成了 User behavior sequence，那么推荐系统的基本问题结构是否已经发生了变化？

在继续查阅论文之后，我逐渐意识到一个重要事实：

渐进式演进并没有改变推荐系统的基本问题结构。

推荐系统仍然是在学习：
Score = f(User, Item, Context)

真正发生变化的是 用户表示方式（User Representation）。
用户不再只是一个静态的 ID embedding，而是由 用户行为序列 经过深度模型编码得到的兴趣表示。
即User不再只是一个ID，而是User history sequence，然后通过序列模型得到用户表示

当把这一过程重新梳理之后，我才逐渐意识到一个更清晰的发展路径：

推荐系统的模型渐进式演化其实经历了一个非常典型的三阶段过程：

  • 静态推荐阶段 用户由 ID embedding 表示，模型通常是简单的 MLP 或特征交叉模型。

  • 序列推荐阶段 用户被表示为行为序列，通过 RNN、Transformer 等模型学习用户兴趣的动态变化。 论文：SASRec

  • Scaling 阶段 在序列模型的基础上进一步扩大模型规模，引入更深的网络结构和更大的参数规模，以提升模型表达能力。 论文：GPTRec

因此，从更宏观的角度看，渐进式演进路径其实包含两个层面的改进：

  1. 模型结构升级：从 MLP 模型逐步发展到序列模型（RNN、Transformer）。
  2. 模型规模扩展（Scaling）：借鉴大模型的发展经验，通过增加模型参数规模提升表达能力。

换句话说，渐进式路径并不是简单地“把模型变大”，而是在保持推荐系统工程架构稳定的前提下，通过 更强的模型结构和更大的模型规模 逐步提升推荐能力。

知识增强路径

在探索第一条 渐进式演进路径 之后，我继续阅读教程中提到的第二条路线：知识增强路径（Knowledge-enhanced Recommendation）。

教程中的描述是：

利用 LLM 蕴含的丰富世界知识，通过语义对齐、特征增强等方式，将 LLM 的语言理解能力注入到传统推荐模型中。

在第一次读到这里时，我其实有些困惑：
如果推荐系统已经可以通过用户行为数据进行学习，那么 为什么还需要引入 LLM 的知识？

进一步思考之后，我了解到：
传统推荐系统虽然在行为建模上非常强，但在 语义理解能力 上其实是非常有限的。

在大多数推荐系统中，物品通常是通过 ID 表示的，例如：
Item ID = 10234

对于推荐模型来说，这只是一个数字。模型可以通过用户行为学习“哪些用户喜欢这个 ID”，但它并不知道这个物品真正代表什么内容。例如：
10234 → iPhone 15

在传统推荐系统中，模型并不会天然理解：

  • iPhone 是一种 智能手机
  • Apple 是一家 科技公司
  • iPhone 和 AirPods 属于同一生态

这些信息其实属于 世界知识（World Knowledge），而大型语言模型正是在这种知识建模上表现非常强。

因此，一些研究开始尝试利用 LLM 来增强推荐系统的语义理解能力。
在这一过程中，我开始理解教程中提到的两个关键概念：

1. 语义对齐（Semantic Alignment）

语义对齐的核心思想是：
让推荐系统中的 物品表示 不再只是 ID，而是与文本语义空间进行对齐。

例如，一个商品通常包含大量文本信息：

  • 商品标题
  • 商品描述
  • 商品属性
  • 用户评论

这些文本可以输入语言模型（如 BERT 或 LLM），得到一个 语义向量表示：

这样一来，推荐系统中的 Item 表示就不再只是一个离散 ID，而是包含了 语义信息的向量表示。模型不仅能够学习用户行为模式，还能够理解内容之间的语义关系，例如：

  • iPhone 与 iPad 属于同一品牌生态
  • DSLR 相机与镜头属于同一摄影领域
  • 科幻电影与太空主题内容具有语义相关性

在阅读这一部分时，我也联想到之前阅读的一篇相关论文 Generative Next POI Recommendation with Semantic ID

在这项研究中，研究者提出了一种 Semantic ID（语义 ID） 的表示方式。
与传统推荐系统直接使用随机数字 ID 不同，Semantic ID 会将物品编码为一组 具有语义结构的 token，例如包含：

  • 物品类别
  • 地理区域
  • 时间模式
  • 协同访问关系

这些语义信息被组合成一个结构化的 ID 序列，使模型能够在推荐过程中利用物品的语义结构进行推理。

从这个角度来看，Semantic ID 可以被理解为一种更进一步的语义对齐方法：
推荐系统不再只是为物品学习一个向量表示，而是尝试构建 具有语义结构的物品表示体系。

2. 特征增强（Feature Augmentation）

另一种思路是利用 LLM 的推理能力，为推荐系统生成新的语义特征。

例如，一个用户的行为记录可能是：

  • iPhone
  • MacBook
  • AirPods

传统推荐系统只看到三个商品 ID。
但语言模型可以推断出一个更高层的语义特征：
User interest = Apple ecosystem

这种语义信息原本并不存在于用户行为数据中，但可以通过 LLM 推理得到，然后作为新的特征加入推荐模型。

因此，如果用一句话总结知识增强路径，它的核心思想其实是：

在传统推荐系统的基础上，引入大型语言模型的语义理解能力，使推荐系统不仅能够学习用户行为模式，还能够理解内容本身的语义信息。

完全生成式路径

在理解知识增强路径之后，我逐渐意识到一个非常有意思的变化。

在最早的推荐系统中，模型主要依赖的是 用户行为数据。推荐系统通过用户点击、浏览、购买等行为信号来学习用户兴趣，本质上是一种 行为驱动（behavior-driven） 的建模方式。

而在知识增强路径中，推荐系统开始引入 语义信息（semantic knowledge）。通过语言模型对文本内容进行建模，系统不仅可以看到用户行为，还能够理解物品内容的语义信息，以及不同内容之间的概念关系。

在这一阶段，推荐系统真正同时拥有了两种重要的信息来源：

  • 行为信息（Behavior Signal）：用户历史交互数据
  • 语义信息（Semantic Knowledge）：由语言模型提供的内容理解能力

当这两类信息被同时纳入推荐模型之后，一个新的问题自然就出现了：

如果推荐系统既能够理解 用户行为序列，又能够理解 内容语义结构，那么推荐问题是否可以被重新定义？

换句话说，推荐系统是否可以不再依赖传统的“召回 → 排序 → 重排”流水线，而是像语言模型生成文本一样，直接生成推荐结果？

从这个角度来看，知识增强路径其实已经为下一阶段的研究奠定了基础：
当 语义信息与行为信息开始被统一建模 时，推荐系统也就逐渐具备了走向 完全生成式推荐（Fully Generative Recommendation） 的条件。
